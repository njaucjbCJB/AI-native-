# 第一阶段前端运行时与开源组件选型

第一阶段继续使用现有 Vite、React、TypeScript 和 Vitest 工程，不迁移到 Create T3 App，也不以 ToolJet 或 Appsmith 作为产品底座。业务蓝图和动态表单采用版本化 JSON 文档与 JSON Schema 契约，使用 AJV 执行结构和数据校验，使用 RJSF 解释并渲染动态表单，使用 XState 表达项目审计审批状态机，使用 shadcn/ui 构建控制台组件；所有 JSON 数据仍通过 StorageAdapter 保存到 LocalStorage，UI 不直接访问存储。Temporal、Langflow、React Flow 和真实数据库保留为后续阶段的可替换实现：只有在出现后端、长时间耐久流程、真实 Agent 编排或可视化流程编辑需求时再引入，避免第一阶段承担与业务验证无关的部署和迁移成本。
