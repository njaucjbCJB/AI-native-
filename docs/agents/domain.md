# Domain Docs

This repo uses a single-context domain documentation layout.

Engineering skills should first read the root project docs when they need domain context:

- `AGENTS.md`
- `PRD.md`
- `TECH_SPEC.md`
- `NEXT_STEPS.md`
- `HANDOFF.md`

If `CONTEXT.md` or `docs/adr/` are added later, skills should also read them when relevant.

If those files do not exist yet, proceed silently and use the existing project docs.
