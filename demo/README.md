# Demo

这是 AI 企业运营框架控制台的可运行 demo，技术栈是 Vite + React + TypeScript + Vitest。

请在这个 `demo/` 目录里运行 npm 命令。如果你在项目根目录运行 `npm test` 或 `npm run build`，npm 会找不到正确的 `package.json` 脚本。

## 安装依赖

```bash
npm install
```

这个命令会安装 demo 需要的前端依赖。第一次运行项目时需要执行一次；依赖文件变化时也需要再次执行。

## 运行测试

```bash
npm test
```

这个命令会运行 Vitest 测试，验证 Blueprint、RequestInstance、WorkflowRuntime、Agent Activity、Reports 和控制台视图模型等核心行为。

## 构建

```bash
npm run build
```

这个命令会运行 TypeScript 编译和 Vite 生产构建，用来确认 demo 可以生成静态网页产物。

## 本地启动

```bash
npm run dev
```

这个命令会启动本地开发服务器。终端会显示一个地址，例如 `http://localhost:5173/`。用浏览器打开后，可以测试：

- 左侧 8 个控制台视图。
- Runtime 采购申请提交。
- 审批角色切换、批准和拒绝。
- Reports 指标和 AI CEO 摘要。
- Agent Activity 调用记录。
- 项目审计自动配置：从中文业务描述生成并部署项目审计 Blueprint。
- 项目台账：查看或维护 3 个示例项目。
- 审计周期：创建周期、选择项目、启动周期、启动后追加项目、关闭周期。
- 项目审计实例：负责人填写和自审批、VP 通过或驳回、AI CEO 生成建议并最终批准或驳回。
- 证据视图：查看项目快照、当前表单值、字段变更记录、审批历史和 AI CEO 建议。
- 重置演示数据：第一次点击进入确认态，第二次点击才会清空演示数据并恢复初始 Blueprint 和 3 个示例项目。

## 项目审计完整演示路径

1. 打开“项目审计自动配置”，点击“生成草稿”，再点击“部署为新版本”。
2. 打开“项目台账”，确认已有 3 个示例项目，也可以编辑项目数据。
3. 打开“审计周期”，创建一个周期，选择项目后点击“启动周期”。
4. 在“项目审计实例”中切换到项目负责人，保存表单、提交并自审批。
5. 切换到分管 VP，点击“VP 通过”；或者填写审批意见后点击“VP 驳回”，再让负责人修改并重新提交。
6. 切换到 AI CEO，点击“生成 AI 建议”，再点击“AI CEO 批准”或填写意见后驳回。
7. 切换到全局查看者，检查已批准实例、审批历史、变更记录和 AI CEO 建议。
8. 需要重新演示时，点击页面右上角“重置演示数据”，再点击“确认重置”。

## 不要提交

不要提交：

- `node_modules/`
- `dist/`
- `.env`
- `.env.*`

这些文件和目录已经由项目 `.gitignore` 排除。
